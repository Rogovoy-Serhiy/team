Типи даних JavaScript та структури даних

Динамическая типизация

JavaScript є слабо типізованою або динамічною мовою. Це означає, що вам не потрібно визначати тип змінної заздалегідь. Тип автоматично визначиться під час виконання програми. Також це означає, що ви можете використовувати одну змінну для зберігання різних типів даних

Типи даних

Стандарт ECMAScript визначає 8 типів:

  • Undefined (Невизначений тип): typeof instance === "undefined".
  • Boolean (Булев, Логічний тип) : typeof instance === "boolean"
  • Number (Число) : typeof instance === "number"
  • String (Рядок) : typeof instance === "string"
  • BigInt typeof instance === "bigint"
  • Symbol (в ECMAScript 6) : typeof instance === "symbol"
  • Null (Null тип) : typeof instance === "object". Спеціальний примітив, що використовується не тільки для даних але і як покажчик на фінальну точку в Ланцюжку Прототипов
  • Object (Объект) : typeof instance === "object". Простая структура, используемая не только для хранения данных, но и для создания других структур, где любая структура создаётся с использованием ключевого слова new: new Object, new Array, new Map (en-US), new Set, new WeakMap, new WeakSet, new Date и множество других структур

І тут нам необхідно зробити застереження щодо використання оператора типуз визначення типу структур,т.к. всі структури повертатимуть "object" при його використанні, оскільки призначення typeof - перевірка типу даних, але не структур. Якщо перевірити тип структури все ж таки необхідно, то в цьому випадку бажано використовувати оператор встанови, оскільки саме він відповідає на питання про те, який конструктор був використаний для створення структури.

Варто відзначити два особливі випадки роботи оператора typeof: повернення "object" для значення null і "function" для функцій: перше прийнято вважати помилкою мови, збереженої для зворотної сумісності, друге є умовністю, зручною для перевірки на належність значення категорії функцій, де функція - це особливий, що "викликається", об'єкт.
